Om de overgang naar een systeem met meerdere rekeningen vlekkeloos te laten verlopen zonder de huidige stabiliteit aan te tasten, voeren we de volgende wijzigingen door: ### Wat is er gewijzigd en waarom? 1. **Data-architectuur (Backward Compatibility):** * Bestaande transacties in `localStorage` hebben nog geen `accountId`. We vangen dit op door een fallback te gebruiken: als een transactie geen ID heeft, wijzen we deze virtueel toe aan de eerste beschikbare rekening. Zo verdwijnen er geen gegevens. 2. **`activeAccountId` Variabele:** * We introduceren een globale variabele die bijhoudt welke rekening momenteel bekeken wordt. Bij het openen van de app staat deze standaard op de eerste rekening. 3. **UI Feedback:** * In de CSS is `.account.active` toegevoegd. Hierdoor ziet de gebruiker welke rekening geselecteerd is. 4. **Filter Logica:** * De functie `render()` filtert nu de lijst op basis van de `activeAccountId` voordat de rijen worden getekend. 5. **Nieuwe transacties:** * De `saveTransaction` functie is aangepast zodat een *nieuwe* transactie automatisch het ID van de actieve rekening meekrijgt. --- ### Volledige Code (Stap 1: Rekeningen klikbaar & Filteren) ```html Payit
